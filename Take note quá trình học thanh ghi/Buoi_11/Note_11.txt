Buổi 11 — TIM1 update interrupt + NVIC (LED PC13)

- main_buoi11_tim1.c
+ Mục tiêu: TIM1 tràn ~0.5s --> vào ISR toggle PC13 (LED Blue Pill, active-low).
+ Chuỗi ĐẦY ĐỦ mà 1 interrupt ngoại vi cần, thiếu 1 khâu là câm:
	RCC clock -> PSC/ARR -> DIER.UIE -> CR1.CEN -> NVIC line -> ISR
+ Clock 8MHz HSI, không PLL. Chu kỳ tràn = (PSC+1)*(ARR+1)/TIM1CLK
	= 8000 * 500 / 8e6 = 0.5s --> LED nháy ~1Hz.
+ BUG GÀI SẴN (cố tình): quên enable NVIC line cho TIM1_UP (IRQ #25)
	--> phía timer đúng hết: timer VẪN tràn (SR.UIF=1 kẹt cứng), CNT vẫn chạy
	--> nhưng NVIC không chuyển request cho core --> ISR không bao giờ chạy, LED tối thui
	--> điển hình "compile ok mà chạy không ra gì" của interrupt.
+ Fix (chip chưa có register map cho NVIC nên ghi thẳng địa chỉ):
	*((volatile unsigned long*)0xE000E100) |= (1UL << 25);	//ISER0 bit25 = TIM1_UP
+ Trong ISR PHẢI clear flag, không thì nó re-fire mãi mãi:
	TIM1.SR.BITS.UIF = 0;

- capture.mac (C-SPY macro, chạy lúc debug — không phải code nạp vào chip)
+ dumpRegs() đọc thanh ghi qua SWD --> runtime_buoi11.log (dàn ST-Link V2 không có UART/SWO)
+ Bằng chứng TRƯỚC fix: NVIC_ISER0 bit25 = 0, TIM1_SR UIF kẹt 1 --> thủ phạm là NVIC.
+ Sau fix: NVIC_ISER0=0x02000000, TIM1_SR=0x1E (UIF đã được ISR clear),
	GPIOC_ODR nhảy 0x2000 <--> 0x0000 giữa 2 lần capture --> LED đang blink thật.

- test.ewp (không copy vào đây, chỉ ghi lại)
+ chỉ đổi đúng 1 dòng: main.c --> main_buoi11_tim1.c (chọn file đem build).
+ test.ewd là config debug local (đường dẫn macro + SWO), không commit.
