Second DMA of Buoi_12: rung 0_1 (HAL) of Problem B — the on-theme one. TIM1 overflows
every 400 ms and raises a DMA *request* (DIER.UDE — not an interrupt); DMA1_Channel5
(hard-wired to TIM1_UP, RM0008 Table 78) writes the next word of a 2-word pattern
{0x2000, 0x0000} into GPIOC->ODR; CIRC=1 reloads CNDTR forever. The LED keeps blinking
WITH THE CORE HALTED at a breakpoint — the strongest "CPU not involved" proof, and the
CNDTR countdown (2->1->reload 2) is finally slow enough to watch, which Problem A never
allowed. Both green cells of the 0_1 row are now PASS on board.

Code / build:
- main_buoi12_dma.c: PROBLEM_A/PROBLEM_B label pair (same #ifdef mechanism as
  STAGE_0_x). Stage01_LedByTimDma(): HAL_TIM_Base_Init (PSC=7999, ARR=399 -> 1 kHz
  tick, beat each 400 ms) + HAL_DMA_Init/Start (MEM->PERIPH, word, MINC=1/PINC=0,
  CIRCULAR, N=2) + __HAL_TIM_ENABLE_DMA(UDE) + HAL_TIM_Base_Start. Debug trick:
  __root pointer constants g_dbg_dma5/g_dbg_tim1 make the macro-peripherals
  watchable in C-SPY Live Watch without being dead-stripped.
- test.ewp: registers stm32f1xx_hal_gpio.c + stm32f1xx_hal_tim.c into the Vendor_HAL
  group for config DMA_B12_S01_HAL (cross-<excluded> kept; the LL `Debug` build is
  untouched). test.ewd stays a working-tree "M" per KI-IAR-0006.

Run (ST-Link V2, pure SWD, C-SPY) — PASS. Fingerprint: CCR5=0x0AB1
(DIR|CIRC|MINC|word|EN, MEM2MEM=0), CNDTR5 bouncing 2<->1, CPAR5=0x4001100C
(GPIOC_ODR), CMAR5=0x20000000 (g_led_pattern), TIM1: CR1.CEN=1, DIER=0x100 (UDE only,
