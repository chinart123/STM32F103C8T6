Two documentation artifacts closing out the vendor-warehouse work:

- docs/detected-issues/.../vcs-git/KI-GIT-0005__github-zip-empty-submodule-folders/
  New KB entry (scope: universal, status: mitigated — recognized BEFORE being
  hit). Documents the two chained traps of bringing someone else's repo into
  your own: (1) GitHub's "Download ZIP" ships submodule folders EMPTY —
  STM32CubeF1 keeps its entire driver payload in 22 submodules, evidenced by
  the verbatim clone log in error.log; (2) a cloned tree carries nested .git
  metadata, which makes the parent git treat it as an embedded repository and
  refuse to track its contents at any nesting depth. Fixes applied and
  verified: clone --recursive --depth 1 --shallow-submodules, recursive .git
  deletion (0 left), 10,653 files tracked, 53 renames recorded. KB README
  catalogue gains the row.
