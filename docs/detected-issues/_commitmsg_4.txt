§4 docs(detected-issues): add KI-GIT-0003 — stale remote URL after GitHub rename

New known-issue entry under a new origin category `vcs-git` (Git/GitHub version
control), promoted from a local scratch explainer.

KI-GIT-0003: after the GitHub repo was renamed 108 -> 103, the local `.git/config`
remote URL stayed on the old name (108). Pushes kept working only because GitHub
redirects the old name ("This repository moved…"), which hid the mismatch and
carries a latent risk (the redirect breaks if the old name is ever reclaimed).
Root cause: renaming a repo on GitHub does NOT update any local clone's
`.git/config`. Fixed with `git remote set-url origin …103….git`; fetch now clean.
