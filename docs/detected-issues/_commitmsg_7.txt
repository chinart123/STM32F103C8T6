§7 docs(detected-issues): add KI-ENV-0004 — C-SPY macro type-mismatch + decimal radix

New known-issue entry under `host-environment` (a local host script tripping over the
IAR C-SPY macro language). Two defects in our own `capture.mac`, surfaced while
capturing TIM1 registers over SWD in the Buoi 11 lesson:

Bug A — `if (f == 0)` raised `capture.mac(48,11): Error: Type mismatch.` and aborted the
macro. Root cause: `__openFile` returns a C-SPY FILE-HANDLE value, not an int; comparing
it to `0` with `==` is a type mismatch. Because `__openFile("a")` had already created the
file, the macro left a 0-byte `runtime.log` — which read like "no output" but was really
"opened, then died before writing". Fix: `if (f) { … } else { … }` (boolean handle test).

Bug B — `__fmessage` printed registers in DECIMAL (`TIM1_SR=31`) because that is the
default radix. Fix: read into a var and format hex, `__fmessage f, "NAME=0x", v:%x, "\n";`.
Post-fix capture confirmed on-board: `NVIC_ISER0=0x02000000`, `TIM1_SR=0x1E`.
