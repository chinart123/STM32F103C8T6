§8 docs(platform): adopt platform v2 blueprint — Markdown-spine reports + figure standards

New `docs/platform-v2.md`: the approved design (user-reviewed 2026-07-08) for how this
repo works as a cross-model AI platform. Key points captured in the blueprint:

- Session reports are Markdown-first (YAML frontmatter + Vietnamese body), generated by
  a stdlib assembler script and indexed one-line-per-report; .docx remains only as a
  frozen hand-off export bridge. Reports optimize for the real reader: any AI model.
- A 3-layer figure discipline for report figures: engines (TikZ/Tectonic, matplotlib,
  WaveDrom), a plain-Markdown style contract (mandatory legend, left-to-right signal
  flow, scientific notation, colorblind-safe palette), and an exemplar-first library
  with a mandatory render-then-self-inspect loop before any figure reaches review.
- A `scope:` portability field for the detected-issues KB (universal | toolchain |
  mcu-specific | process) so lessons carry over cleanly when the project moves to a
  new MCU/IDE, aligned with the comeback branching strategy (eternal main + era tags).
- Governance invariants B1–B5 (clean CV-facing HEAD, paired main<->note commits per
  session, immovable source/note folder paths, separate edit/commit/push approvals,
  single-home rules with pointers elsewhere).
