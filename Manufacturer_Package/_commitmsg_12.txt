§12 feat(vendor): seed No.0_C&C++_Industrial_Draft — move counterpart files into C&C++ mirror

Rearranges 53 vendor files (names and contents 100% untouched — pure `git mv`)
from the STM32CubeF1 warehouse into `Manufacturer_Package/No.0_C&C++_Industrial_Draft/`,
a structural mirror of the hand-written firmware tree `C&C++/`:

- 25 x .c  -> Embedded_C99/Microcontroller/stm32f10x/driver/
- 26 x .h  -> Embedded_C99/Microcontroller/stm32f10x/driver/include/
- stm32f1xx_hal.c / stm32f1xx_hal_def.h -> 0_common/ (role twin of common.c/common.h)

Mapping rule (user-approved): a vendor file moves in ONLY if the hand-written
tree already has a counterpart driver — gpio, tim (fsm_time), rcc (+ll_bus),
exti, flash, i2c, pwr, rtc, spi, uart (both hal_uart async + hal_usart sync,
since the hand-written uart.h represents the whole USART peripheral). Files
with no counterpart yet stay in the warehouse: CMSIS, cortex/NVIC, ADC — and
deliberately DMA (stm32f1xx_hal_dma / ll_dma), which will move in the very
commit where the DMA lesson creates its hand-written counterpart. From here on,
`git log --follow` on any Draft file dates exactly when study reached it — the
raw data for the knowledge-progress chart in future session reports.

Build note: Draft is a package warehouse, not an IAR project. When vendor-track
code is compiled later, include paths must cover BOTH Draft and the warehouse
(HAL sources still need CMSIS device headers + a hal_conf, which have no
counterpart yet) — no linker surprises by design.
